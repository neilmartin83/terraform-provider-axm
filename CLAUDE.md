## What this is

Terraform provider for the Apple School and Business Manager (ASM/ABM) API. Built on `terraform-plugin-framework` v1 (protocol v6). Provider type name is `axm`.

## Commands

```sh
make build        # go build -v ./...
make install      # build + go install
make fmt          # gofmt -s -w -e .
make lint         # golangci-lint run
make generate     # cd tools; go generate ./... (regenerates docs and code)
make test         # go test -v -cover -timeout=120s -parallel=10 ./...
make testacc      # TF_ACC=1 go test -v -cover -timeout 120m ./...
```

Single test: `go test ./internal/... -run '^TestName$'`

Acceptance tests require `AXM_*` env vars — see `.env` for the shape. Source it with `. .env` before running.

## Architecture

**Entry point:** `main.go` → `internal/provider/provider.go`

**internal/provider** — schema/config, registers all resources/data sources/list resources, sets up the tflogger.

**internal/client** — the API client:
- `client.go`: HTTP transport with retry logic (max 5 retries; 429 waits on `Retry-After` ≤60s, otherwise errors; 502/503/504 uses exponential backoff)
- `client_oauth.go`: JWT client assertion generation + OAuth2 token exchange against `account.apple.com`. Assertions and tokens are cached to disk under `~/.axm/cache` and reused across runs.
- One file per API domain (`blueprints.go`, `configurations.go`, `mdm_servers.go`, `org_devices.go`, etc.)

**internal/common** — shared helpers used across all resource packages:
- `configure.go`: `ConfigureClient()` — extracts `*client.Client` from provider data; call this in every resource/data source `Configure` method
- `filters.go`: `NormalizedFilterString()` — trims and validates filter inputs
- Additional timeout and type conversion helpers

**internal/resources** — one package per Terraform construct. Full-CRUD resources (blueprint, configuration, device_management_service) use this file split:

| File | Purpose |
|---|---|
| `resource.go` | Schema, Metadata, Configure, ImportState, IdentitySchema |
| `crud.go` | Create, Read, Update, Delete |
| `model_types.go` | Go structs for API request/response bodies |
| `schema_types.go` | Terraform schema attribute types |
| `helpers.go` | Internal helpers for the package |
| `list_resource.go` | List resource implementation (if applicable) |
| `data_source.go` | Data source implementation (if applicable) |

Read-only constructs (app, apps, audit_events, organization_device, packages, users, user_groups, etc.) only have `data_source.go` and `model_types.go`.

**docs/** — generated by `make generate`; do not edit manually.

**testing/** — manual Terraform configs and exploratory Go scripts; not part of the test suite.

## Key conventions

- Schema rule: **sets** for user-supplied unordered collections; **lists** for computed API data.
- Scope: `education.api` (ASM) vs `business.api` (ABM). Use `client.IsBusinessScope()` to gate business-only behaviour.
- Keep dependencies to native Go, `golang.org/x`, and Terraform Plugin Framework packages only.
- Copyright header on every Go file: `// Copyright Neil Martin 2026` + `// SPDX-License-Identifier: MPL-2.0`
- Comments: short one-liners on exported identifiers only; no comments inside function bodies or type definitions.
